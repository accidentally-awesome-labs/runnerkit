<objective>
Wire the provisioned cloud machine into the already-proven BYO runner lifecycle and expose provider identity through operations commands.

Purpose: Phase 4 should not create a second runner installer. Once Hetzner readiness passes, RunnerKit must use the same GitHub registration token timing, bootstrap, online verification, labels, status, logs, and doctor patterns already implemented for BYO, while adding visible provider facts and drift warnings.

Output: Cloud `runnerkit up` install path through shared BYO bootstrap, final cloud state persistence, provider facts in status/doctor/logs human and JSON output, and tests proving no operational command mutates resources.
</objective>
