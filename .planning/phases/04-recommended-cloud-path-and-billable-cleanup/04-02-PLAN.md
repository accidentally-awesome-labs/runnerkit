<objective>
Implement the safe Hetzner provisioning and readiness workflow for the recommended cloud machine path.

Purpose: Developers need RunnerKit to create the low-cost cloud host reliably, capture every provider resource identity needed for later status/destroy, and avoid split-brain or orphaned billable resources if readiness fails.

Output: Hetzner SDK adapter/fake client tests, deterministic resource naming/tags, SSH key/firewall/server creation, provider inventory in state, pending checkpoints after billable resource creation, and readiness gates before GitHub registration.
</objective>

```go
type ProvisionResult struct {
    Machine Machine

}
```
