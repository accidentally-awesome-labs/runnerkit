<interfaces>
Provider package contracts to create in `internal/provider/provider.go`:

```go
type Provider interface {
    Name() string
    Validate(ctx context.Context, input ProvisionInput) (ValidationResult, error)
    Plan(ctx context.Context, input ProvisionInput) (ProvisionPlan, error)
    Provision(ctx context.Context, input ProvisionInput) (ProvisionResult, error)
    WaitReady(ctx context.Context, machine Machine) (Machine, error)
    Describe(ctx context.Context, ref state.ProviderRef) (ProviderStatus, error)
    Destroy(ctx context.Context, ref state.ProviderRef) (DestroyResult, error)
    VerifyDestroyed(ctx context.Context, ref state.ProviderRef) (VerificationResult, error)
}
```

Default profile constants:

```text
provider=hetzner
region=fsn1
server_type=cpx22
image=ubuntu-24.04
ssh_user=runnerkit-admin
ssh_allowed_cidr=0.0.0.0/0
estimated_hourly_cost=approximate static catalog value
estimated_monthly_cost=approximate static catalog value
cost_estimate_caveat=Estimated cost is approximate. Provider pricing varies by region and time; billing stops only after RunnerKit-created billable resources are deleted or verified non-billable.
```

CLI flags to add to `runnerkit up`:

```text
--cloud string            recommended cloud provider; only hetzner is supported in Phase 4
--cloud-region string     provider region/location; default fsn1
--cloud-profile string    provider server profile; default cpx22
--ssh-allowed-cidr string SSH ingress CIDR; default 0.0.0.0/0
```

GitHub token-timing contract:

```text
Before cloud provider mutation, RunnerKit may resolve repository metadata, safety, and non-mutating runner-management read access, but it must not call CheckRunnerManagementPermission, VerifyAuth, CreateRegistrationToken, or any method that mints a GitHub runner registration token.
After cloud readiness and BYO preflight pass, RunnerKit may mint the just-in-time registration token and continue through the shared bootstrap path.
```

Required UI copy from UI-SPEC:

```text
Choose setup path for `owner/name`:
Use existing SSH host (BYO)
Provision recommended cloud runner (Hetzner)
RunnerKit will not create billable cloud resources without an explicit `--cloud hetzner` flag and `--yes`.
Cloud runner provisioning plan
Estimated cost is approximate. Provider pricing varies by region and time; billing stops only after RunnerKit-created billable resources are deleted or verified non-billable.
Provision cloud runner
No RunnerKit-managed cloud runner is saved for `owner/name`.
Run `runnerkit up --repo owner/name --cloud hetzner` to create one, or pass `--host user@host` to use an existing machine.
Create billable Hetzner resources for `owner/name`?
Future cleanup: `runnerkit destroy --repo owner/name`
```
