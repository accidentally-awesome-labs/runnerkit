<objective>
Add guided recovery for common RunnerKit-managed BYO persistent runner failures.

Purpose: Developers should be able to restart, reinstall, or re-register a stopped/offline managed runner without blindly rerunning the full setup flow, while preserving host-key safety and token redaction.

Output: Recovery plan model, `runnerkit recover` command, service restart/reinstall flow, re-registration flow with fresh tokens, state update/checkpoints, tests, and docs for common recovery cases.
</objective>

<interfaces>
Add this command:

```text
runnerkit recover --repo owner/name --dry-run
runnerkit recover --repo owner/name --restart-service --yes
runnerkit recover --repo owner/name --reinstall-service --yes
runnerkit recover --repo owner/name --reregister --yes
```

Flags: `--repo owner/name`, `--yes`, `--dry-run`, `--restart-service`, `--reinstall-service`, and `--reregister`. Global `--json` and `--no-color` remain inherited.

Recovery remote command IDs:

- `recover.service.restart`
- `recover.service.verify`
- `recover.service.reinstall`
- `recover.service.stop`
- `recover.runner.remove`
- `recover.runner.configure`
- `recover.runner.start`

JSON top-level keys: `ok`, `command`, `repo`, `dry_run`, `plan`, `results`, `state_updated`, `github_runner_id`, and `redactions_applied`.
</interfaces>
