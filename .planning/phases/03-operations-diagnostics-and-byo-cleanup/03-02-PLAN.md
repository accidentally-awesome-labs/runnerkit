<objective>
Add read-only operational evidence commands: `runnerkit logs` and `runnerkit doctor`.

Purpose: Developers should inspect recent runner/service evidence and get actionable findings without SSH spelunking or leaking tokens, while keeping automatic repair out of scope.

Output: Bounded redacted log collection, stable doctor findings/remediations, JSON contracts, read-only tests, redaction regressions, and docs that promote status/logs/doctor as the first troubleshooting path.
</objective>

Add these commands and contracts:

```text
runnerkit logs --repo owner/name --since 1h --lines 200
runnerkit doctor --repo owner/name
```

`runnerkit logs` flags: `--repo owner/name`, `--since` default `1h`, `--lines` default `200`, `--service`, and `--runner`. Global `--json` and `--no-color` remain inherited.

`runnerkit doctor` flags: `--repo owner/name` and `--verbose`. There is no `--fix` flag in Phase 3.

```go
// internal/ops/logs.go
const (
    CommandLogsSystemdJournal = "logs.systemd.journal"
    CommandLogsRunnerDiagList = "logs.runner.diag.list"
    CommandLogsRunnerDiagTail = "logs.runner.diag.tail"
)
type LogSection struct { Source, Title, Metadata, Content string; Warnings []string }
```

```go
// internal/ops/doctor.go
type Finding struct { ID, Severity, Source, Evidence, Remediation string }
type DoctorReport struct { Repo, StatePath string; Health Health; Findings []Finding; NextActions []NextAction }
```
