<objective>
Build the shared Phase 3 operations/status foundation and expose `runnerkit status`.

Purpose: Developers need one fast, safe answer to "is this managed BYO runner ready, and what should I run next?" before deeper logs, doctor, recovery, and cleanup commands can reuse the same facts.

Output: Reusable testsupport fakes, state helpers, fast remote status probes, a stable observed-runner/health model, and a read-only `runnerkit status` command with human and JSON contracts.
</objective>
