<objective>
Implement GitHub repository targeting, least-privilege auth discovery, permission checks, runner-token adapter fixtures, and public/fork-risk safety gates for `runnerkit up`.

Purpose: RunnerKit cannot become a safe runner management tool unless it can identify the correct repository, verify credentials with minimum required permissions, fail closed with remediation, and avoid leaking short-lived registration/removal tokens. This plan implements D-05, D-06, D-07, D-08, and the GitHub-facing pieces of GH-01.

Output: A tested `internal/github` adapter layer and `runnerkit up` integration that resolves repo targets, reuses `gh` auth first, explains fine-grained token fallback, checks runner-management capability, gates public repositories, and redacts token responses.
</objective>
