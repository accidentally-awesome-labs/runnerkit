<objective>
Close the Phase 1 verification gaps by replacing the production fake-permitted GitHub default with real auth, repository metadata, and runner-management permission checks, and by giving the production binary an OS command runner for `gh` and `git remote` subprocesses.

Purpose: Phase 1 cannot be marked complete while `runnerkit up` can save or preview success without a real GitHub credential, can force repository metadata to private, or cannot execute git remote auto-detection in the actual binary.

Output: A surgical production wiring fix with deterministic injected fakes in tests only. This plan does not add BYO runner installation, cloud provisioning, cleanup, diagnostics, or ephemeral mode.
</objective>

<interfaces>
Use the existing CLI-facing interface without widening `runnerkit up` scope:

```go
// internal/cli/up.go
type GitHubService interface {
    Repository(ctx context.Context, repo github.Repo) (github.Repo, error)
    VerifyAuth(ctx context.Context, repo github.Repo) (github.PermissionStatus, error)
}
```

Add a real implementation in `internal/github`:

```go
// internal/github/service.go
type ServiceOptions struct {
    CommandRunner CommandRunner
    Env           map[string]string
    BaseURL       string
    HTTPClient    *http.Client
    Redactor      *redact.Redactor
}

func NewService(opts ServiceOptions) *Service
func (s *Service) Repository(ctx context.Context, repo Repo) (Repo, error)
func (s *Service) VerifyAuth(ctx context.Context, repo Repo) (PermissionStatus, error)
```

`Service.VerifyAuth` must call `DiscoverAuth` and `CheckRunnerManagementPermission`. `Service.Repository` must call `Client.Repository`. Both methods must use the discovered token in memory only.
</interfaces>
