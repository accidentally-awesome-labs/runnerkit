<objective>
Implement the scoped ephemeral runner lifecycle: one-job registration, bounded TTL/finalizer behavior, state metadata, log preservation, operations semantics, and cleanup integration.

Purpose: Developers who explicitly choose ephemeral mode need a real RunnerKit-managed runner that GitHub will assign at most one job, while RunnerKit preserves troubleshooting evidence and prevents cloud/BYO cleanup confusion. This is not autoscaling, ARC, JIT, or a fleet manager.

Output: Bootstrap scripts using `config.sh --ephemeral`, custom one-shot systemd unit, finalizer and TTL timer scripts, ephemeral BYO/cloud `up` paths, provider/state metadata, mode-aware status/logs/doctor output, and log preservation before `down`/`destroy` cleanup.
</objective>

Ephemeral bootstrap command contract:

```text
./config.sh --unattended --url https://github.com/owner/name --token "$RUNNERKIT_REGISTRATION_TOKEN" --name <runner> --labels <labels> --work <workdir> --replace --ephemeral
```

Ephemeral remote paths and units:

```text
install_path=/opt/actions-runner/<runner>
work_dir=/var/lib/runnerkit/work/<runner>
log_archive_path=/var/lib/runnerkit/ephemeral/<runner>/logs
finalizer_path=/usr/local/lib/runnerkit/ephemeral/<runner>/finalize.sh
service_name=runnerkit-ephemeral.<runner>.service
ttl_service_name=runnerkit-ephemeral.<runner>.ttl.service
ttl_timer_name=runnerkit-ephemeral.<runner>.ttl.timer
default_ttl=24h
```

Ephemeral systemd unit requirements:

```text
Restart=no
ExecStart=<installPath>/run.sh
ExecStopPost=<finalizerPath> completed
```

Finalizer/log preservation requirements:

```text
copy _diag/Runner_*.log into /var/lib/runnerkit/ephemeral/<runner>/logs
copy _diag/Worker_*.log into /var/lib/runnerkit/ephemeral/<runner>/logs
write bounded journalctl output to /var/lib/runnerkit/ephemeral/<runner>/logs/systemd-journal.log
write sentinel /var/lib/runnerkit/ephemeral/<runner>/state.json
remove local .runner .credentials .credentials_rsaparams best-effort
never include registration/removal/provider token values in scripts, state, stdout, or stderr
```
