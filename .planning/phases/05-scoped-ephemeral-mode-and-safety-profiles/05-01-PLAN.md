<objective>
Add the explicit mode/profile selection and safety-policy foundation for persistent versus ephemeral runners.

Purpose: Phase 5 starts by making mode choice understandable before mutation. Developers must see the cost, isolation, cleanup, operational, and log-preservation tradeoffs, and must be steered away from persistent self-hosted runners for public/fork/untrusted workloads.

Output: Mode and safety-profile model, `runnerkit up --mode persistent|ephemeral`, default `--ephemeral-ttl 24h`, interactive mode/profile options, persistent and ephemeral label/snippet support, updated public/fork safety copy, human/JSON tradeoff output, and tests proving side-effect timing.
</objective>

Mode flags to add to `runnerkit up`:

```text
--mode persistent|ephemeral
--ephemeral-ttl duration    default 24h
--allow-ephemeral-byo-risk  allow risky public/fork BYO ephemeral setup only with --mode ephemeral --host and explicit acknowledgement
```

Mode constants and safety profile constants:

```go
const (
    ModePersistent = "persistent"
    ModeEphemeral = "ephemeral"

    ProfilePersistentTrusted = "persistent-trusted"
    ProfilePersistentRisky = "persistent-risky"
    ProfileEphemeralBYO = "ephemeral-byo"
    ProfileEphemeralCloud = "ephemeral-cloud"

    DefaultEphemeralTTL = 24 * time.Hour
)
```

Required mode/profile prompt copy from UI-SPEC:

```text
Choose runner mode for `owner/name`:
Persistent trusted runner
Reuses one runner across trusted private jobs. Lowest ongoing friction, but unsafe for public, fork, or untrusted workflows.
Ephemeral one-job runner on existing machine
GitHub assigns one job then deregisters the runner. The machine is reused, so this is not a clean VM.
Ephemeral one-job cloud runner (Hetzner)
Stronger isolation for risky workloads. Creates billable resources until `runnerkit destroy` verifies cleanup.
Default mode: persistent for trusted private repositories.
Ephemeral mode: one GitHub job, automatic deregistration, TTL cleanup, and preserved troubleshooting logs.
Ephemeral mode is not a fleet manager. RunnerKit creates one scoped runner; jobs with matching labels can still queue if no runner is online.
```

Required safety copy from UI-SPEC:

```text
Persistent self-hosted runners are unsafe for public, fork-based, or otherwise untrusted workflows.
Use `runnerkit up --repo owner/name --mode ephemeral --cloud hetzner` for stronger isolation, or use GitHub-hosted runners.
Only pass `--allow-public-repo-risk` if you accept that untrusted code can execute repeatedly on your machine.
BYO ephemeral mode is a one-job GitHub registration, not a clean virtual machine. Do not store unrelated secrets on the host.
Ephemeral cloud runners still create billable Hetzner resources. Billing stops only after `runnerkit destroy --repo owner/name` verifies cleanup.
```

Required snippets:

```yaml
runs-on: [self-hosted, runnerkit, runnerkit-owner-repo, linux, x64, persistent]
runs-on: [self-hosted, runnerkit, runnerkit-owner-repo, linux, x64, ephemeral]
```
